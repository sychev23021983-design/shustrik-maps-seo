# Шаг 1 — быстрые технические правки (готово к внедрению)

## Порядок действий

1. **redirects_301_safe.csv** — 4 явных дубля карточек (одинаковый слаг + "-2").
   Импортировать в плагин Redirection (WordPress) ИЛИ вставить `htaccess_redirects.txt`
   в .htaccess до блока `# BEGIN WordPress`.

2. **noindex_junk_pages.csv** — 39 служебных страниц (Prepayment/test).
   Варианты внедрения (выбрать один):
   - Если есть Yoast/RankMath — проставить noindex вручную/массово через их интерфейс по списку slug
   - Либо вставить functions_snippet_noindex.php в functions.php темы (или в плагин "Code Snippets")
   - Дополнительно: добавить строки из robots_txt_additions.txt в robots.txt — это не
     заменяет noindex, а экономит краулинговый бюджет (страницы не будут сканироваться зря)

3. **cannibalization_manual_review.csv** — 18 пар карточек, которые НЕ трогать
   автоматически. Нужно открыть каждую пару вручную, сравнить контент и решить:
   объединить (редирект слабой на сильную) или развести по разным намерениям.
   Редиректить их бездумно нельзя — это разные карточки с разным контентом.

## После внедрения
- Проверить в Google Search Console → Sitemaps, что 39 junk-страниц пропали из индекса
  (может занять 1-4 недели)
- Проверить редиректы через curl -I или любой redirect-checker
- Отметить в STATUS.md / audit-log.md репозитория, что Шаг "быстрые правки" выполнен
