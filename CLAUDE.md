# CLAUDE.md — инструкции для Claude Code в этом репозитории

Это репозиторий SEO-проекта для shustrik-maps.com (магазин 3D-моделей и карт: STL для
3D-печати/CNC, OBJ/FBX/C4D для визуализации, GeoTIFF/PSD/SVG для GIS/дизайна).

## Первое, что нужно сделать в любой сессии
Прочитай `STATUS.md` полностью — там текущее состояние проекта, что сделано, что в работе,
и roadmap следующих шагов. Не пересказывай пользователю то, что уже написано в STATUS.md,
как будто это новое — используй это как контекст.

## Структура данных
- `semantic-core/categories.csv` — 30 категорий сайта с готовыми SEO-полями
- `semantic-core/products_seo.csv` — 517 товарных карточек с SEO-полями и статусом
  (OK / DUPLICATE / RENAME / JUNK/EXCLUDE)
- `semantic-core/duplicates_issues.csv` — найденные дубли/каннибализация с рекомендациями
- `semantic-core/junk_pages.csv` — служебные страницы на noindex
- `lsi-keyword-bank.csv` — LSI-слова по типу карточки

## Правила работы
- Все данные читай из CSV через pandas, не пересчитывай классификацию заново без необходимости.
- При добавлении новых товаров/категорий — придерживайся уже принятой таксономии
  (Group → Category → Format), не изобретай новую структуру.
- В конце сессии: обнови `STATUS.md` (дату наверху, разделы "Сделано"/"В работе") и добавь
  короткую запись в `audit-log.md` с датой и сутью изменений.
- Не удаляй исторические записи из `audit-log.md` — это журнал решений, а не черновик.
