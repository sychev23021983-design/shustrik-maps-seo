# Audit Log

История решений и изменений по SEO-проекту shustrik-maps.com.
Формат: дата — что сделано — кем/почему.

---

## 2026-07-07 — Первичный аудит и семантическое ядро
- Собран полный список карточек сайта (517 шт.) через site-map-html (стр. 1-2) и структура
  из 30 категорий через меню сайта.
- Проведена классификация по типу карточки (STL/CNC, визуализация, спутник, ночной спутник,
  relief-legacy, без воды, elevation, историческая, outline, planet, fantasy, здания,
  minecraft, art/novelty).
- Найдено 39 служебных страниц (Prepayment-*, test) — рекомендация: noindex.
- Найдено ~22 группы дублирующихся/конкурирующих карточек (одна страна+тип — 2 URL).
- Сгенерированы SEO Title/H1/Meta/Primary keyword/Secondary keywords/LSI для всех
  категорий и товарных карточек.
- Ограничения: нет данных по частотности/сложности запросов (нужен внешний источник),
  ~6 карточек с нестандартными названиями требуют ручной проверки geo-парсинга.

## 2026-07-07 — Шаг 2: архитектура сайта и меню
- Диагностированы структурные проблемы: общий URL категории "Fantasy Maps" для двух веток
  (3D Models и CNC Models); терминологическая путаница "3D Maps" vs "3D Models"; отсутствие
  отдельных категорий для гор/озёр/вулканов.
- Спроектирована двухслойная архитектура: существующая навигация по формату (3D Models /
  CNC Models / Maps) сохраняется без изменений; добавляется новый слой — Geo Hub Pages
  (одна страница на гео-объект, объединяющая ссылки на все доступные форматы) для закрытия
  сильных гео-запросов ("Portugal terrain model" и т.п.) одной страницей вместо нескольких
  конкурирующих карточек.
- Предложена URL-схема: `/countries/{country}/`, `/countries/{country}/stl/`,
  `/countries/{country}/3d-model/`, `/countries/{country}/satellite-map/` и аналогично для
  us-states, canadian-provinces, national-parks, continents, islands, cities, а также новых
  категорий mountains/lakes/volcanoes.
- Согласован порядок внедрения: сначала разбор дублей/junk-страниц и категории Fantasy,
  затем пилот geo-хабов на топ-30 странах, затем масштабирование и новые категории.
- Статус: архитектура спроектирована, ни одна geo-хаб страница ещё не создана на сайте.

## 2026-07-07 — Пакет быстрых технических правок (Шаг 1, low-risk часть)
- Разделены 22 группы дублей на две категории: 4 явных дубля одной карточки (одинаковый
  слаг с "-2") — безопасны для автоматического 301-редиректа; 18 пар — каннибализация
  (разные карточки, разный контент), помечены для ручного review, редиректить нельзя.
- Подготовлены готовые к внедрению файлы: noindex_junk_pages.csv, redirects_301_safe.csv,
  htaccess_redirects.txt, cannibalization_manual_review.csv, functions_snippet_noindex.php,
  robots_txt_additions.txt.

## 2026-07-07 — Переход на единый xlsx вместо CSV
- Владельцу сайта неудобно работать с несколькими CSV-файлами — перешли на единый рабочий
  файл `semantic-core/shustrik-maps_SEO_audit_and_core.xlsx` (5 листов: README, Categories,
  Products_SEO, Duplicates_Issues, LSI_Keyword_Bank).
- В листы Categories и Products_SEO добавлены колонки Applied? (Yes/No/Skipped, выпадающий
  список), Date Applied, Notes — для отслеживания прогресса внедрения по каждой
  категории/карточке. Строки с Applied=Yes подсвечиваются зелёным, Skipped — серым.
- Старые CSV в semantic-core/ и отдельный lsi-keyword-bank.csv удалены из репозитория —
  весь семантический core теперь только в этом xlsx.
- Ограничение: xlsx — бинарный формат, git не показывает построчный diff при коммитах
  (в отличие от CSV), но версии сохраняются и файл читается нормально в новых сессиях.
